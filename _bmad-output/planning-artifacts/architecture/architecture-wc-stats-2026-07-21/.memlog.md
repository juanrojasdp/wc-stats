- (decision) AD-8 [ADOPTED]: pipeline correctness paradigm — fail loud per report, batch continues; binary per-report Self-Validation (exact marker count + 100 percent link rate); run manifest is the single record of truth; assert-on-unknown for RGB keys and anchors; unlinked markers retained+flagged, never dropped. - (decision) AD-9: pipeline staging — per-report Extract phase emits a persisted Extraction Record (work/extracted/{match-id}.json, raw domains, self-validation result); global Precompute phase (identity resolution, aggregation, bundle+index emission) is the only stage with cross-report knowledge. Prevents: aggregation logic creeping into extractors; re-parsing PDFs to tweak precompute.
