### The AD-14 flow (verbatim recipe, `contract/README.md` ~lines 419–442)

1. Epic 2 requests the change, naming the surface and the field.
2. Epic 1 implements it in `/contract`.
3. A numbered logged decision is added to `contract/README.md` (format: `### N. <one-line title>` + rationale + guarding test; 16 exist, next is 17).
4. `schemaVersion` bumped in `contract/version.json` (single global integer, declared exactly once).
5. Fixtures AND generated types regenerated in the SAME commit — "the step that gets skipped under pressure". Guards: `pipeline/tests/test_fixtures.py`, `pipeline/tests/test_contract_schemas.py::test_the_committed_generated_types_still_match_the_schemas`, `npm run check:types`.

"Sample set" = the AD-8 stratified template-consistency sample (≥1 report per venue and per matchday round). The gate: this sign-off lands before Epic 1 extraction proceeds past that sample into the full 104 batch. Sequencing interlock per implementation-readiness report: `1.1 → (2.1..2.3 ∥ 1.2..1.5) → 2.3 sign-off → 1.6+` (1.6 already landed; 1.7+ are what this gate actually still holds).
